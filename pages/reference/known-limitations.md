# 既知の制限（Known Limitations）

現時点の River Reviewer は、OSS としての基盤整備を進めている段階です。利用時の注意点をまとめます。

- ローカル実行（`river run`）は「プレビュー/検証」用途が中心である（`--dry-run` や API キー未設定時のフォールバックあり）。
- スキルはサンプルと実運用想定のものが混在する。必要に応じてプロジェクト向けに調整すること。
- 破壊的変更が入り得るため、GitHub Actions はリリースタグ（例: `@v0.1.0`）へのピン留めを推奨する。

ロードマップは `ROADMAP.md` を参照してください。
