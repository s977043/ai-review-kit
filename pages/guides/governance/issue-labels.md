# Issue 管理（テンプレート / ラベル）

River Reviewer では、Issue のトリアージと優先度付けをしやすくするために、テンプレートとラベルを併用します。

## Issue テンプレート

`Issues → New issue`（または [issues/new/choose](https://github.com/s977043/river-reviewer/issues/new/choose)）から作成してください。

## ラベルの目安

### 優先度

- `P0`: 最優先（リリースや致命的不具合など）
- `P1`: 高（近いリリースで対応したい）
- `P2`: 中（余裕があれば対応）

### ロードマップ

- `Phase 1` / `Phase 2` / `Phase 3` / `Backlog`

### 種別（type）

Issue テンプレートでは、以下の形式のラベルを基本にしています。

- `type:bug`
- `type:feature`
- `type:enhancement`
- `type:docs`
- `type:content`
- `type:task`

### ステータス（status）

- `status:triage`: まずは状況確認中

## OSS ラベル

- `good first issue`: 初めての貢献に適した Issue
- `help wanted`: 協力者を歓迎する Issue
