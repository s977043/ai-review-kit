# チェックの作り方

AI Review Kitでは、独自のチェックを追加できる。チェックを記述する際は以下の点に注意する。

- 各チェックは1つの観点にフォーカスする（例: セキュリティ、パフォーマンス、可読性）。
- 出力は「短い見出し」「説明」「修正案」の3つで構成する。
- YAMLまたはJavaScriptON形式の設定ファイルで登録する。

以下に簡単な例を示す:

```yaml
id: no-hardcoded-secret
title: ハードコードされた証言情報の検出
description: コードにハードコードされたトークンやパスワードがないか検査する。
severity: critical
```
