# example-2: upstream のみ（設計/ドキュメント変更向け）

upstream フェーズのスキルだけを動かすサンプルです。

## できること

- 設計・ドキュメント変更（`docs/` など）に対して upstream レビューを実行

## セットアップ

1. このディレクトリを対象リポジトリにコピーします（`.github/workflows/` を含む）
2. `docs/` 配下を変更する PR を作成します

## 補足

- upstream のスキルは、`applyTo` が `docs/` / `design/` などに寄っているため、コード変更だけのPRだと候補が少ない場合があります。
