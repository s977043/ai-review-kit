# コードレビュー観点チェックリスト

## セキュリティ

- [ ] 外部入力のバリデーションがあり、危険な文字列をそのまま使っていない
- [ ] 認可チェックが欠落していない（特に新規 API や管理系操作）
- [ ] 秘密情報・トークン・鍵が平文で露出していない

## 性能

- [ ] 重い処理や N+1 の兆候がない、必要ならバッチ化・キャッシュ化されている
- [ ] 大きなデータのコピーや不要な再描画など無駄な計算がない

## 品質

- [ ] 責務や名前が明確で、ガード節で読みやすくなっている
- [ ] ログと例外処理の方針が周囲と揃っている
- [ ] マジックナンバーやハードコード URL が定数化されている

## テスト

- [ ] 新規/変更コードに対応するテストが追加・更新されている
- [ ] モックの粒度が適切で、フレークの原因が仕込まれていない
- [ ] 失敗パスや境界値がカバーされている
